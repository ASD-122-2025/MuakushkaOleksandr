Мета: Ознайомитися з аналізом складності алгоритмів на прикладі методів сортування — вставками та обміну (бульбашкою).¶

1. Теоретичний аналіз асимптотичної складності алгоритмів сортування: метод вставки та метод бульбашки.¶

Сортування методом вставки:¶

У найкращому випадку (масив вже відсортований) маємо складність O(n), оскільки кожен елемент порівнюється лише один раз.¶

У найгіршому випадку (масив впорядкований у зворотному порядку) складність зростає до O(n²), бо кожен новий елемент вставляється на початок, проходячи через усі попередні.¶

Асимптотична оцінка:¶

T(n) = (n−1)(c₁ + c₂ + c₆) + ((n(n−1))/2)(c₃ + c₄ + c₅), де домінує n².¶

2. Сортування методом бульбашки:¶

Алгоритм неефективний для великих наборів даних через велику кількість порівнянь.¶

У найгіршому (і найтиповішому) випадку складність становить O(n²):¶

T(n) = (n−1)·c₁ + ((n(n−1))/2)·c₂ + c₃ + c₄ — домінуючим є n².¶

2. Реалізація алгоритму сортування методом бульбашки на Python¶

def bubble_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break
    return arr

4. Контрольні питання.¶

1. О-нотація це верхня межа асимптотичної складності алгоритму, а Ω-нотація це нижня межа асимптотичної складності алгоритму.¶

2. Складність для цього метода буде O(n), оскільки ми вставляємо відразу в початок масиву, тож нам не потрібно проходити через кожен елемент поки ми не знайдемо правильне положення для елемента, це означає що цикл вставки виконується лише один раз.¶

3. Ефективним алгоритмом є той, що за найменший час може опрацювати найбільшу кількість елементів.¶

4. Цей параметр характеризує кількість вхідних даних і використовується для аналізу складності алгоритму.¶

5. O(n^3). Тому що n^3 є домінуючим членом цієї функції (константи відкидаємо та менші члени також).¶
